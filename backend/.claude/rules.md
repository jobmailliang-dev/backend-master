# 系统介入规则

当 AI 需要用户介入决策时，应发送系统通知。通知应包含以下要素：

## 介入场景

在以下情况下，AI 应请求用户介入：

1. **安全相关决策**
   - 涉及代码执行、文件修改、系统命令
   - 需要访问敏感信息或配置
   - 执行可能影响系统稳定性的操作

   2. **架构/设计决策**
      - 改变项目整体架构
      - 添加新的依赖或外部服务
      - 修改核心业务逻辑

3. **需要确认的操作**
   - 删除、覆盖重要文件
   - 大规模重构
   - 不可逆的操作

4. **用户偏好相关**
   - 不确定用户意图时
   - 有多种实现方案时
   - 需要用户输入特定信息时

## 通知格式

当需要介入时，使用以下格式通知用户：

```
[介入请求]
类型: {安全|架构|确认|信息请求}
描述: {简要说明当前情况}
选项: {可选方案列表}
```

## 响应方式

用户可以通过以下方式响应：

- 提供额外信息或选择
- 授权继续操作
- 要求修改方案
- 取消当前操作
